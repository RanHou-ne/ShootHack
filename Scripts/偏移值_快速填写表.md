# 🎯 偏移值快速填写表

> **使用本表：** 在 dump.cs 中找到对应的值，填入 `GOM_Frida_Script.js` 顶部的常数定义中

---

## 📍 需要填写的偏移值

| 常数名 | 说明 | dump.cs搜索关键词 | 预期值范围 | 你的值 |
|------|------|----------|----------|------|
| **TAG_OFFSET** | GameObject 的 tag 字符串指针偏移 | `public string tag { get; }` (在 GameObject 类) | 0x50 ~ 0x70 | `0x____` |
| **TRANSFORM_OFFSET** | GameObject 的 Transform 指针偏移 | `public Transform transform { get; }` (在 GameObject 类) | 0x20 ~ 0x40 | `0x____` |
| **POS_OFFSET** | Transform 的 position 浮点数起始偏移 | `public Vector3 position { get; }` (在 Transform 类) | 0x28 ~ 0x38 | `0x____` |
| **GOM_OFFSET** | GameObjectManager 的全局地址偏移 | `s_Instance` 或 CheatEngine 找 | 任意 | `0x________` |
| **UPDATE_OFFSET** | Update 函数的相对地址（用于 Hook） | 任意 `Update()` 的 RVA | 任意 | `0x________` |

## 🔍 快速查找步骤

### 第 1 步：找 GameObject 类（约第 147765 行）
```bash
# PowerShell
Select-String -Path dump.cs -Pattern "public sealed class GameObject" | Select-Object -First 1
```

### 第 2 步：复制相关段落到记事本
找到后，复制该类的前 100 行，找出以下字段的偏移注释：
```
public Transform transform { get; }    // 0x__ ← 填入 TRANSFORM_OFFSET
public string tag { get; }             // 0x__ ← 填入 TAG_OFFSET
```

### 第 3 步：找 Transform 类
```bash
Select-String -Path dump.cs -Pattern "public class Transform.*Component"
```

在 Transform 类中找：
```
public Vector3 position { get; set; }  // 0x__ ← 填入 POS_OFFSET
```

### 第 4 步：找 GameObjectManager（或用 CheatEngine）
```bash
Select-String -Path dump.cs -Pattern "GameObjectManager|gameObjects|s_Instance" | head -20
```

### 第 5 步：填写到脚本中
打开 `Scripts/GOM_Frida_Script.js`，修改顶部：
```javascript
const TAG_OFFSET = 0x___;              // 替换为第 1 步找到的值
const TRANSFORM_OFFSET = 0x___;        // 替换为第 2 步找到的值
const POS_OFFSET = 0x___;              // 替换为第 3 步找到的值
const GOM_OFFSET = 0x__________;       // 替换为第 4 步找到的值
const UPDATE_OFFSET = 0x__________;    // 替换为任意 Update 的 RVA
```

## 📝 现有项目代码中已确认的信息

从 `src/Engine/il2cpp_function.h` 可知，项目已在使用以下 IL2CPP 函数：

```c++
// GameObject 方法
DO_FUNC(void*, GameObject_get_transform, (void*), ..., "get_transform");
DO_FUNC(void*, GameObject_get_tag, (void*), ..., "get_tag");

// Transform 方法
DO_FUNC(Unity_Vector3, Transform_get_position, (void*), ..., "get_position");
```

**这意味着：** tag 和 position 在内存中是可直接访问的属性，对应的偏移应该是存在的。

## 🎮 临时测试方案（如果找不到 GOM_OFFSET）

如果你实在无法从 dump.cs 找到 GOM_OFFSET，可以：

1. **使用项目现有的 `FindObjectsOfType()` API**
   - 编辑 `src/Engine/il2cpp_function.h`，添加需要的方法
   - 在 C++ 代码中调用 `FindObjectsOfType()` 遍历对象

2. **用 CheatEngine 动态扫描**
   - 附加进程，搜索已知的 Player GameObject 地址
   - 反推 activeObjects vector 的位置
   - 计算 GOM_OFFSET
