## 一、整体架构概览

本项目的核心目标：**注入 DLL 到 Unity IL2CPP 游戏进程，遍历所有 GameObject，读取其世界坐标，投影到屏幕坐标后绘制指针地址文字**。

整个数据流如下：

```
DLL注入
  └─ DllMain
       ├─ Engine::Initialize()          ← 绑定所有 IL2CPP API + 游戏 C# 方法指针
       └─ DX11HOOK()                    ← 劫持 DX11 SwapChain 虚表
            └─ hkPresent()（每帧调用）
                 └─ ESP::Render()
                      └─ DrawAllObjectPointers()
                           ├─ il2cpp_thread_attach()       ← 线程安全
                           ├─ GetMainCameraSafe()          ← 获取主摄像机
                           ├─ FindObjectsOfType()          ← 获取所有 GameObject
                           ├─ for each GameObject:
                           │    ├─ GetTransform()          ← 获取 Transform 组件
                           │    ├─ GetPosition()           ← 读取世界坐标
                           │    ├─ WorldToScreen()         ← 3D→2D 投影
                           │    └─ ImGui::AddText()        ← 屏幕绘制
                           └─ il2cpp_thread_detach()       ← 线程分离
```

涉及的核心模块和文件：

| 模块 | 文件 | 职责 |
|:---|:---|:---|
| **引擎层** | `src/Engine/Engine.h/cpp` | IL2CPP API 绑定、GetMethod、工具函数 |
| **引擎层** | `src/Engine/il2cpp_api.h` | IL2CPP 官方 C API 列表 |
| **引擎层** | `src/Engine/il2cpp_function.h` | 游戏 C# 方法列表（X-Macro 数据表） |
| **引擎层** | `src/Engine/il2cpp_types.h` | Il2CppObject、数组布局等核心结构 |
| **引擎层** | `src/Engine/class.h/cpp` | Camera、Transform、GameObject 等游戏类型 |
| **Hook 层** | `src/Hook/DX11Hook.cpp` | DX11 Present Hook，每帧渲染入口 |
| **ESP 层** | `src/ESP/ESP.h/cpp` | ESP 绘制功能，遍历对象+屏幕绘制 |
| **核心层** | `src/Core/Struct.h` | Vector2/3、D3DXMATRIX 等基础类型 |

## 二、前置知识：IL2CPP 运行时层次结构

在 IL2CPP 中，所有类型信息按以下层次组织。**理解这个层次是理解整个流程的基础**：

```
Domain（运行时域，全局唯一）
  │
  ├── Assembly（程序集 = 一个 .dll 文件）
  │     │
  │     └── Image（镜像 = 程序集的元数据视图）
  │           │
  │           └── Class（类 = C# 中的类型定义）
  │                 │
  │                 ├── Method（方法）
  │                 ├── Field（字段）
  │                 └── Property（属性）
  │
  ├── Assembly（另一个程序集）
  │     └── ...
  └── ...
```

对应的 IL2CPP C API 调用链：

```
il2cpp_domain_get()                          → Domain*
il2cpp_domain_assembly_open(domain, name)     → Assembly*
il2cpp_assembly_get_image(assembly)           → Image*
il2cpp_class_from_name(image, ns, className)  → Class*
il2cpp_class_get_method_from_name(cls, name)  → MethodInfo*
*(void**)MethodInfo                           → 函数指针
```

## 三、完整流程详解（6大阶段）

### 阶段1：引擎初始化 — 函数指针绑定

**入口**：`Engine::Initialize()`（在 `src/Engine/Engine.cpp` 中）

DLL 注入后必须首先调用此函数，完成所有函数指针的绑定。绑定分为两类：

#### 1.1 IL2CPP 官方 C API 绑定（GetProcAddress）

IL2CPP 导出函数名固定为 `il2cpp_xxx`，可以通过 `GetProcAddress` 直接获取：

```cpp
// Engine.cpp - Initialize() 内部，X-Macro 展开点 ③
#define DO_API(RetType, Name, Args) \
    Name = reinterpret_cast<Name##_t>(GetProcAddress(g_engineGameAssembly, #Name));

#include "il2cpp_api.h"   // ← 展开所有 GetProcAddress 绑定
#undef DO_API
```

展开效果示例：

```cpp
// il2cpp_api.h 中声明：
// DO_API(Il2CppDomain*, il2cpp_domain_get, ())

// 展开为：
il2cpp_domain_get = reinterpret_cast<il2cpp_domain_get_t>(
    GetProcAddress(g_engineGameAssembly, "il2cpp_domain_get"));
```

**原理**：Windows 加载 DLL 时，导出函数名和地址记录在 PE 头导出表中，`GetProcAddress` 在表中按名称查找返回地址。

#### 1.2 游戏 C# 方法绑定（GetMethod）

C# 方法编译后名称混淆且不导出，必须通过 IL2CPP 元数据遍历定位：

```cpp
// Engine.cpp - Initialize() 内部，X-Macro 展开点 ④
#define DO_FUNC(RetType, Name, Args, AssemblyName, Namespace, ClassName, MethodName) \
    Name = reinterpret_cast<Name##_t>(GetMethod(AssemblyName, Namespace, ClassName, MethodName));

#include "il2cpp_function.h"   // ← 展开所有 GetMethod 绑定
#undef DO_FUNC
```

展开效果示例：

```cpp
// il2cpp_function.h 中声明：
// DO_FUNC(Unity_Array*, FindObjectsOfType, (void*),
//         "UnityEngine.CoreModule", "UnityEngine", "Object", "FindObjectsOfType")

// 展开为：
FindObjectsOfType = reinterpret_cast<FindObjectsOfType_t>(
    GetMethod("UnityEngine.CoreModule", "UnityEngine", "Object", "FindObjectsOfType"));
```

#### 1.3 GetMethod 内部实现

```cpp
// src/Engine/Engine.cpp
void* GetMethod(const char* AssemblyName,
                const char* Namespace,
                const char* ClassName,
                const char* MethodName)
{
    // 1. 获取 Domain（最顶层容器）
    Il2CppDomain* pDomain = il2cpp_domain_get();

    // 2. 打开程序集（如 "UnityEngine.CoreModule"）
    const Il2CppAssembly* pAssembly = il2cpp_domain_assembly_open(pDomain, AssemblyName);

    // 3. 获取 Image（程序集的元数据视图）
    const Il2CppImage* pImage = il2cpp_assembly_get_image(pAssembly);

    // 4. 根据命名空间+类名找到 Class
    Il2CppClass* pClass = il2cpp_class_from_name(pImage, Namespace, ClassName);

    // 5. 在类中查找方法（-1 表示忽略参数个数）
    const MethodInfo* pMethod = il2cpp_class_get_method_from_name(pClass, MethodName, -1);

    // 6. 从 MethodInfo 提取函数指针
    //    MethodInfo 第一个字段就是 methodPointer，用 *(void**) 解引用头部 8 字节
    void* methodPtr = *(void**)pMethod;
    return methodPtr;
}
```

**为什么用 `*(void**)pMethod`？**

`MethodInfo` 结构体的第一个成员是函数指针：

```cpp
struct MethodInfo {
    void* methodPointer;    // 偏移 0x00，就是我们要的函数地址
    Il2CppClass* klass;     // 偏移 0x08
    // ... 其他元数据
};
```

因为 `MethodInfo` 只有前向声明（没有完整定义），不能写 `pMethod->methodPointer`，只能用指针解引用取头部 8 字节。

#### 1.4 il2cpp_function.h 中声明方法的规则

```cpp
// DO_FUNC 参数说明：
// DO_FUNC(返回类型, C++变量名, (参数列表), "程序集名", "命名空间", "类名", "C#方法名")

// 实例方法：第一个参数写 (void*) 代表 this 指针
DO_FUNC(void*, GameObject_get_transform, (void*),
        "UnityEngine.CoreModule", "UnityEngine", "GameObject", "get_transform");

// 静态方法：直接写方法参数
DO_FUNC(Unity_Array*, FindObjectsOfType, (void*),
        "UnityEngine.CoreModule", "UnityEngine", "Object", "FindObjectsOfType");

// 返回值类型为值类型时，用完整类型定义
DO_FUNC(Unity_Vector3, Transform_get_position, (void*),
        "UnityEngine.CoreModule", "UnityEngine", "Transform", "get_position");
```

**C# 属性的 IL2CPP 方法名规则**：

| C# 属性 | IL2CPP 方法名 |
|:---|:---|
| `transform { get; }` | `"get_transform"` |
| `layer { get; set; }` | `"get_layer"` / `"set_layer"` |
| `position { get; }` | `"get_position"` |

### 阶段2：DX11 Hook — 渲染循环劫持

**入口**：`DX11HOOK()`（在 `src/Hook/DX11Hook.cpp` 中）

DX11 Hook 的目标是劫持游戏的渲染循环，使我们的代码在每帧绘制时执行。

#### 2.1 Hook 安装流程

```cpp
bool DX11HOOK()
{
    // 1. 查找 Unity 窗口
    g_hWnd = FindWindowA("UnityWndClass", nullptr);

    // 2. 创建临时 SwapChain 获取虚函数表
    D3D11CreateDeviceAndSwapChain(..., &tmpSC, ...);
    g_vtableSwapChain = *reinterpret_cast<DWORD64**>(tmpSC);

    // 3. 记录原始函数指针
    g_oPresent = reinterpret_cast<PresentFn>(g_vtableSwapChain[8]);   // Present 在虚表第8项
    g_oResize  = reinterpret_cast<ResizeFn>(g_vtableSwapChain[13]);   // ResizeBuffers 在第13项

    // 4. 替换虚表项为我们自己的函数
    VirtualProtect(g_vtableSwapChain, ..., PAGE_EXECUTE_READWRITE, &protect);
    g_vtableSwapChain[8]  = reinterpret_cast<DWORD64>(hkPresent);   // 替换 Present
    g_vtableSwapChain[13] = reinterpret_cast<DWORD64>(hkResize);    // 替换 ResizeBuffers
    VirtualProtect(...);
}
```

#### 2.2 每帧渲染入口 hkPresent

```cpp
HRESULT WINAPI hkPresent(IDXGISwapChain* This, UINT SyncInterval, UINT Flags)
{
    // 首次调用时初始化 ImGui
    if (!g_imguiState.dx11BackendInitialized) { /* 初始化 ImGui */ }

    // 每帧渲染
    if (g_imguiReadyForRender && g_mainRenderTargetView)
    {
        ImGui_ImplDX11_NewFrame();
        ImGui_ImplWin32_NewFrame();
        ImGui::NewFrame();

        // ★ ESP 绘制入口 ★
        ESP::Render();   // ← 每帧调用，遍历对象并绘制

        ImGui::Render();
        ImGui_ImplDX11_RenderDrawData(ImGui::GetDrawData());
    }

    return g_oPresent(This, SyncInterval, Flags);  // 调用原始 Present
}
```

### 阶段3：线程附加 — IL2CPP 线程安全

**关键**：DX11 Present Hook 运行在游戏渲染线程上，但 IL2CPP GC 不知道这个线程的存在。如果不 `attach`，调用 `FindObjectsOfType` 等 GC 相关函数时可能导致崩溃。

#### 3.1 旧写法（手动 attach/detach，已废弃）

```cpp
// ❌ 旧写法：每个 return 前都要手动 detach，容易遗漏
Il2CppThread* pAttachedThread = nullptr;
if (il2cpp_thread_attach && il2cpp_domain_get)
{
    Il2CppDomain* pDomain = il2cpp_domain_get();
    if (pDomain)
        pAttachedThread = il2cpp_thread_attach(pDomain);
}

// ... IL2CPP 操作 ...
if (!xxx) { if (pAttachedThread && il2cpp_thread_detach) il2cpp_thread_detach(pAttachedThread); return; }
// ↑ 每个 return 都要重复这行

// 结束时必须 detach
if (pAttachedThread && il2cpp_thread_detach)
    il2cpp_thread_detach(pAttachedThread);
```

#### 3.2 新写法（RAII 自动管理，推荐）

```cpp
// ✅ 新写法：Engine::ThreadGuard，构造时自动 attach，离开作用域自动 detach
// 即使中途 return 或异常也不会泄漏
Engine::ThreadGuard guard;
if (!guard.IsValid()) return;

// ... IL2CPP 操作，可以自由 return，无需手动 detach ...
```

**原理**：`ThreadGuard` 使用 RAII（资源获取即初始化）模式，析构函数自动调用 `il2cpp_thread_detach`。

### 阶段4：获取 GameObject 数组 — FindObjectsOfType

这是遍历所有游戏对象的核心步骤，分为两步：构造 `System.Type` 参数、调用 `FindObjectsOfType`。

#### 4.1 构造 System.Type 参数

`FindObjectsOfType` 的 C# 签名是 `static Object[] FindObjectsOfType(Type type)`，需要一个 `System.Type` 实例。

##### 旧写法（手动链式调用，已废弃）

```cpp
// ❌ 旧写法：6步链式调用 + 每步 null 检查 + 每步 detach
Il2CppDomain* pDomain = il2cpp_domain_get();
const Il2CppAssembly* pAssembly = il2cpp_domain_assembly_open(pDomain, "UnityEngine.CoreModule");
if (!pAssembly) { /* detach & return */ }

const Il2CppImage* pImage = il2cpp_assembly_get_image(pAssembly);
if (!pImage) { /* detach & return */ }

Il2CppClass* pClass = il2cpp_class_from_name(pImage, "UnityEngine", "GameObject");
if (!pClass) { /* detach & return */ }

const Il2CppType* pType = il2cpp_class_get_type(pClass);
if (!pType) { /* detach & return */ }

Il2CppObject* pTypeObj = il2cpp_type_get_object(pType);
if (!pTypeObj) { /* detach & return */ }
```

##### 新写法（一行调用，推荐）

项目已封装了两个通用工具函数，消除重复代码：

```cpp
// ✅ 新写法：一行获取 C# System.Type 实例
Il2CppObject* pTypeObj = Engine::GetTypeObject("UnityEngine.CoreModule", "UnityEngine", "GameObject");
if (!pTypeObj) return;

// 如果只需要 Il2CppClass*（用于字段遍历、方法查找等）：
Il2CppClass* pClass = Engine::GetClass("UnityEngine.CoreModule", "UnityEngine", "GameObject");
```

**封装函数说明**：

| 函数 | 返回值 | 用途 |
|:---|:---|:---|
| `Engine::GetClass(Assembly, Namespace, Class)` | `Il2CppClass*` | 字段遍历、方法查找、DumpClass |
| `Engine::GetTypeObject(Assembly, Namespace, Class)` | `Il2CppObject*` | 给 `FindObjectsOfType` 等 C# 方法传 Type 参数 |

**为什么需要 `il2cpp_type_get_object`？**

`FindObjectsOfType` 需要的是 `System.Type`（C# 层面的反射对象），而不是 `Il2CppType*`（C++ 结构）。`il2cpp_type_get_object` 就是这个转换桥梁。`Engine::GetTypeObject` 内部自动完成了 `GetClass → class_get_type → type_get_object` 的全部调用。

#### 4.2 调用 FindObjectsOfType 获取数组

```cpp
// 调用绑定的函数指针
System_Object_array* ObjArray = reinterpret_cast<System_Object_array*>(FindObjectsOfType(pTypeObj));

// 数组结构（定义在 il2cpp_types.h 中）：
// struct System_Object_array {
//     Il2CppObject        obj;         // 对象头（klass + monitor）
//     Il2CppArrayBounds*  bounds;      // 多维数组时非空，一维为 nullptr
//     il2cpp_array_size_t max_length;  // ★ 元素总数
//     Il2CppObject*       m_Items[1];  // ★ 柔性数组，实际长度 = max_length
// };

size_t total = static_cast<size_t>(ObjArray->max_length);
```

**数组遍历的内存原理**：

```
System_Object_array 内存布局：
┌─────────────────────────────────────────────────┐
│ +0x00  Il2CppObject obj   (klass + monitor)     │  ← 16 字节对象头
│ +0x10  Il2CppArrayBounds* bounds                │  ← 8 字节
│ +0x18  il2cpp_array_size_t max_length           │  ← 8 字节，元素总数
│ +0x20  Il2CppObject* m_Items[0]                 │  ← 第 1 个元素指针
│ +0x28  Il2CppObject* m_Items[1]                 │  ← 第 2 个元素指针
│ +0x30  Il2CppObject* m_Items[2]                 │  ← 第 3 个元素指针
│ ...                                             │
└─────────────────────────────────────────────────┘
```

```cpp
// il2cpp_function.h 中的声明
DO_FUNC(void*, GameObject_get_transform, (void*),

```

注意：实例方法的第一个参数是 `this` 指针，所以参数列表是 `(void*)`。

#### 5.3 获取世界坐标

```cpp
// Engine.cpp 中的封装
Unity_Vector3 GetPosition(void* pTransform)
{
    if (!pTransform || !Transform_get_position)
        return { 0, 0, 0 };
    return Transform_get_position(pTransform);  // 调用绑定的 C# 方法指针
}
```

`Unity_Vector3` 是值类型（struct），直接通过函数返回值获取：

```cpp
// class.h 中的定义
struct Unity_Vector3
{
    float x;
    float y;
    float z;
};
```

**为什么返回值类型用 `Unity_Vector3` 而不是 `Unity_Vector3*`？**

值类型在 C# 中是内联存储的，IL2CPP 编译后的 C++ 函数通过寄存器/栈直接返回结构体，不返回指针。所以 `DO_FUNC` 中声明为 `Unity_Vector3` 而非 `Unity_Vector3*`。

### 阶段6：3D→2D 投影与屏幕绘制

#### 6.1 WorldToScreen 投影原理

```cpp
// src/Engine/class.cpp
bool Camera::WorldToScreen(Vector3 position, Vector2& Point)
{
    // 1. 从 Camera 对象读取视图投影矩阵
    //    Camera 偏移 +0x10 处有一个指针，间接寻址到矩阵数据
    uint8_t* temp01 = *(uint8_t**)(((uint8_t*)this) + 0x10);
    D3DXMATRIX* Matrix = (D3DXMATRIX*)(temp01 + 0xDC);

    // 2. 齐次裁剪坐标计算
    float Z = Matrix->_14 * position.X + Matrix->_24 * position.Y
            + Matrix->_34 * position.Z + Matrix->_44;

    // 3. Z > 0 表示在摄像机前方（可见）
    if (Z > 0)
    {
        float X = Matrix->_11 * position.X + Matrix->_21 * position.Y
                + Matrix->_31 * position.Z + Matrix->_41;
        float Y = Matrix->_12 * position.X + Matrix->_22 * position.Y
                + Matrix->_32 * position.Z + Matrix->_42;

        // 4. 透视除法 → NDC 归一化设备坐标（范围 -1 到 1）
        float NdcX = X / Z;
        float NdcY = Y / Z;

        // 5. NDC → 屏幕像素坐标
        Point.X = (NdcX + 1.f) / 2.f * ImGui::GetIO().DisplaySize.x;
        Point.Y = (1.f - (NdcY + 1.f) / 2.f) * ImGui::GetIO().DisplaySize.y;
        //        ^ Y 轴翻转：NDC 的 Y 向上，屏幕的 Y 向下

        return true;
    }
    return false;  // 在摄像机后方，不可见
}
```

**投影变换数学原理**：

```
世界坐标 (Xw, Yw, Zw, 1)
       │
       │  × 视图投影矩阵 (ViewProjection Matrix)
       ↓
裁剪坐标 (Xc, Yc, Zc, Wc)
       │
       │  透视除法: NdcX = Xc/Wc, NdcY = Yc/Wc
       ↓
NDC 归一化坐标 (-1~1)
       │
       │  映射到屏幕: ScreenX = (NdcX+1)/2 × 屏幕宽
       │              ScreenY = (1-NdcY)/2 × 屏幕高
       ↓
屏幕像素坐标 (ScreenX, ScreenY)
```

> **注意**：偏移 `+0x10` 和 `+0xDC` 是硬编码的，会随 Unity 版本变化。更稳定的做法是通过 `Camera.GetViewProjectionMatrix()` 等 C# 方法获取矩阵。

#### 6.2 ImGui 屏幕绘制

```cpp
// 将 Unity_Vector3 转换为 Vector3（Types.h 中的类型）

Vector2 screenPos;

// WorldToScreen 投影

{
```

## 四、从 dump.cs 定位所需的类或字段

### 4.1 dump.cs 是什么

`dump.cs` 是 [Il2CppDumper](https://github.com/Perfare/Il2CppDumper) 工具对游戏的 `GameAssembly.dll` 进行逆向分析后自动生成的文件，包含游戏中**所有 C# 类型定义的完整声明**，格式类似 C# 源码。

**生成方法**：

```
1. 下载 Il2CppDumper: https://github.com/Perfare/Il2CppDumper/releases
2. 准备文件:
   - 游戏安装目录/游戏名_Data/il2cpp_data/Metadata/global-metadata.dat
   - 游戏安装目录/游戏名_Data/Plugins/x86_64/GameAssembly.dll（或 libil2cpp.so）
3. 运行 Il2CppDumper，选择以上两个文件
4. 输出目录中会生成 dump.cs
```

**dump.cs 文件结构示例**：


### 4.2 定位类的完整步骤

**场景**：你想找到一个游戏中表示"玩家"的类。

#### 步骤1：搜索关键词

在 dump.cs 中搜索可能的关键词：

| 搜索策略 | 搜索关键词示例 |
|:---|:---|
| 语义搜索 | `Player`、`Character`、`Pawn`、`Actor`、`Unit` |
| 功能搜索 | `Health`、`Damage`、`Weapon`、`Shoot`、`Inventory` |
| 继承搜索 | `: MonoBehaviour`（游戏逻辑类通常继承它） |
| 命名空间搜索 | 游戏特有命名空间（如 `Game.`、`FPS.`） |

#### 步骤2：确认类的完整路径

找到类后，记录四个关键信息（就是 `DO_FUNC` 需要的参数）：

```csharp
// dump.cs 中找到：
namespace GameSpecific           // ← 命名空间
{
    public class Player : MonoBehaviour   // ← 类名
    {
```

提取信息：

| 信息 | 值 | 对应 DO_FUNC 参数 |
|:---|:---|:---|
| 程序集名 | 需要额外确认（见下文） | `AssemblyName` |
| 命名空间 | `"GameSpecific"` | `Namespace` |
| 类名 | `"Player"` | `ClassName` |

#### 步骤3：确认程序集名

程序集名**不一定等于命名空间**。确认方法：

1. **在 dump.cs 顶部查看**：Il2CppDumper 通常在每个类前注释程序集名
2. **使用 Engine::DumpAssembly 运行时确认**：

```cpp
// 打印所有已加载程序集
Il2CppDomain* pDomain = il2cpp_domain_get();
size_t asmCount = 0;
const Il2CppAssembly** asms = il2cpp_domain_get_assemblies(pDomain, &asmCount);
for (size_t i = 0; i < asmCount; i++)
{
    const Il2CppImage* img = il2cpp_assembly_get_image(asms[i]);
    printf("[%zu] %s\n", i, il2cpp_image_get_name(img));
}
```

3. **常见 Unity 程序集对照**：

| 程序集名 | 包含内容 |
|:---|:---|
| `UnityEngine.CoreModule` | Unity 核心类：GameObject, Transform, Object 等 |
| `UnityEngine.PhysicsModule` | 物理相关：Rigidbody, Collider 等 |
| `UnityEngine.IMGUIModule` | UI 相关 |
| `Assembly-CSharp` | **游戏主代码**（大部分自定义类在这里） |
| `Assembly-CSharp-firstpass` | 游戏插件/第三方代码 |

4. **用 `il2cpp_class_get_assemblyname` 运行时验证**：

```cpp
Il2CppClass* pClass = il2cpp_class_from_name(pImage, "GameSpecific", "Player");
if (pClass)
{
    const char* asmName = il2cpp_class_get_assemblyname(pClass);
    printf("Player 类所在的程序集: %s\n", asmName);
}
```

### 4.3 定位字段的完整步骤

**场景**：你想读取 `Player` 类的 `health` 字段。

#### 步骤1：在 dump.cs 中找到字段偏移

```csharp
public class Player : MonoBehaviour
{
    private float health;       // 0x1C    ← 偏移 0x1C
    private float maxHealth;    // 0x20
    private int teamId;         // 0x24
    private bool isDead;        // 0x28
}
```

**偏移的含义**：`health` 字段在 `Player` 对象内存中，相对于对象起始地址偏移 `0x1C` 字节的位置。

#### 步骤2：理解偏移的计算

```
Player 对象内存布局：
┌──────────────────────────────────────────────────────┐
│ +0x00  Il2CppClass* klass       ← 继承自 Il2CppObject│
│ +0x08  void* monitor            ← 继承自 Il2CppObject│
│ +0x10  m_Name (string)          ← 继承自 Object       │
│ +0x18  m_InstanceID (int)       ← 继承自 Object       │
│ +0x1C  health (float)           ← Player 自己的字段   │
│ +0x20  maxHealth (float)        ← Player 自己的字段   │
│ +0x24  teamId (int)             ← Player 自己的字段   │
│ +0x28  isDead (bool)            ← Player 自己的字段   │
└──────────────────────────────────────────────────────┘
```

> dump.cs 中标注的偏移是**相对于对象起始地址**的绝对偏移，已经包含了父类字段的大小。

#### 步骤3：通过代码读取字段值

**方法 A：直接偏移访问（快但脆弱）**

```cpp
float GetPlayerHealth(void* pPlayer)
{
    // 偏移 0x1C 是从 dump.cs 中获取的
    return *reinterpret_cast<float*>((uintptr_t)pPlayer + 0x1C);
}
```

**方法 B：通过 IL2CPP API 动态获取偏移（稳定，推荐）**

```cpp
float GetPlayerHealth(void* pPlayer)
{
    // 1. 找到 Player 类
    Il2CppClass* pClass = il2cpp_class_from_name(pImage, "GameSpecific", "Player");

    // 2. 按字段名查找 FieldInfo
    FieldInfo* pField = il2cpp_class_get_field_from_name(pClass, "health");

    // 3. 获取运行时偏移（不依赖硬编码偏移）
    size_t offset = il2cpp_field_get_offset(pField);
    printf("health 偏移: 0x%zX\n", offset);  // 应输出 0x1C

    // 4. 读取值
    float health = *reinterpret_cast<float*>((uintptr_t)pPlayer + offset);
    return health;
}
```

**方法 C：通过 il2cpp_field_get_value（最安全）**

```cpp
float GetPlayerHealth(void* pPlayer)
{
    FieldInfo* pField = il2cpp_class_get_field_from_name(pClass, "health");
    float health = 0.0f;
    il2cpp_field_get_value(reinterpret_cast<Il2CppObject*>(pPlayer), pField, &health);
    return health;
}
```

#### 步骤4：处理不同类型的字段

| C# 字段类型 | C++ 读取方式 | 示例 |
|:---|:---|:---|
| `int` / `float` / `bool` | 直接解引用 | `*(int*)(obj + offset)` |
| `string` | 读指针→转 UTF-16 | `il2cpp_string_chars(il2cpp_string*)` |
| `引用类型`（class） | 读指针 | `*(void**)(obj + offset)` |
| `数组 T[]` | 读指针→遍历 | `System_Object_array*` 结构 |
| `静态字段` | 从类静态数据区读取 | `il2cpp_class_get_static_field_data` + offset |

#### 步骤1：在 dump.cs 中找到方法

### 4.5 常见陷阱与注意事项

#### 陷阱1：程序集名猜错

```
❌ 错误：DO_FUNC(..., "UnityEngine.CoreModule", "GameSpecific", "Player", ...)
✅ 正确：DO_FUNC(..., "Assembly-CSharp", "GameSpecific", "Player", ...)
```

游戏自定义类通常在 `Assembly-CSharp` 中，不在 `UnityEngine.CoreModule`。

#### 陷阱2：C# 属性方法名与属性名不同

```csharp
public float Health { get; }    // 属性名是 "Health"
                                // 但方法名是 "get_Health"
```

```
❌ 错误：DO_FUNC(..., "Health")
✅ 正确：DO_FUNC(..., "get_Health")
```

#### 陷阱3：字段偏移随版本变化

dump.cs 中的偏移是**特定游戏版本**的。游戏更新后偏移可能改变。

**解决方案**：使用 `il2cpp_class_get_field_from_name` + `il2cpp_field_get_offset` 动态获取偏移。

#### 陷阱4：泛型类型

```csharp
public class List<T> { }     // dump.cs 中可能有 List<int>、List<string> 等
```

泛型实例化类型在 IL2CPP 中的命名空间和类名可能很复杂，需要特殊处理。对于 `FindObjectsOfType` 等需要 `Type` 参数的场景，泛型类型不能直接用 `il2cpp_class_from_name`。

#### 陷阱5：元数据裁剪（Strip）

Unity 的 IL2CPP 构建配置中如果启用了代码裁剪（Managed Stripping Level），部分类/方法可能被裁掉。表现为 `il2cpp_class_from_name` 或 `il2cpp_class_get_method_from_name` 返回 `nullptr`。

**解决方案**：
1. 在 dump.cs 中确认该类/方法存在
2. 运行时用 `Engine::DumpClass` 验证
3. 如果确实被裁掉，只能用内存特征码扫描定位

#### 步骤5：在 ESP.h/cpp 中添加绘制函数

```cpp
// ESP.h
namespace ESP {
    void DrawPlayerHealth();   // 新增
}

// ESP.cpp —— 使用新封装后的简化写法
void DrawPlayerHealth()
{
    if (!MenuSwitches::g_Toggles.DrawPlayerHealth) return;

    // 1. 线程安全（RAII 自动管理，无需手动 detach）
    Engine::ThreadGuard guard;
    if (!guard.IsValid()) return;

    // 2. 获取主摄像机
    Camera* pCamera = Engine::GetMainCameraSafe();
    if (!pCamera) return;

    // 3. 一行获取 Player 的 System.Type 实例
    Il2CppObject* pTypeObj = Engine::GetTypeObject("Assembly-CSharp", "GameSpecific", "Player");
    if (!pTypeObj) return;

    // 4. 调用 FindObjectsOfType 获取所有 Player
    if (!FindObjectsOfType) return;
    System_Object_array* players = reinterpret_cast<System_Object_array*>(FindObjectsOfType(pTypeObj));
    if (!players || players->max_length <= 0) return;

    ImDrawList* drawList = ImGui::GetForegroundDrawList();

    for (size_t i = 0; i < players->max_length; i++)
    {
        void* pPlayer = players->m_Items[i];
        if (!pPlayer) continue;

        // 5. 获取 Transform → 世界坐标
        void* pTransform = Engine::GetTransform(pPlayer);
        if (!pTransform) continue;

        Unity_Vector3 worldPos = Engine::GetPosition(pTransform);

        // 6. 投影到屏幕
        Vector3 pos3D(worldPos.x, worldPos.y, worldPos.z);
        Vector2 screenPos;
        if (pCamera->WorldToScreen(pos3D, screenPos))
        {
            // 7. 读取血量
            float health = Engine::GetPlayerHealth(pPlayer);

            // 8. 绘制血量文字
            char buf[64];
            sprintf_s(buf, "HP: %.0f", health);
            drawList->AddText(
                ImVec2(screenPos.X, screenPos.Y - 15),
                IM_COL32(0, 255, 0, 255),
                buf
            );
        }
    }
    // guard 离开作用域，自动 detach，无需手动处理
}
```
