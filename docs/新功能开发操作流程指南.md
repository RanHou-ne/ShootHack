## 一、功能类型分类

在开始之前，先判断你要做的功能属于哪一类，不同类型需要修改的文件不同：

| 功能类型 | 说明 | 需要改的核心文件 |
|:---|:---|:---|
| **ESP 绘制类** | 在屏幕上画方框/名字/血量/骨骼/射线等 | `il2cpp_function.h` → `class.h` → `Engine.h/cpp` → `MenuSwitches.h` → `ESP.h/cpp` → `Menu.cpp` |
| **内存修改类** | 修改游戏内存值（血量/弹药/速度等） | `il2cpp_function.h` → `class.h` → `Engine.h/cpp` → `MenuSwitches.h` → `Menu.cpp`（逻辑在 Engine 层） |
| **UI 纯展示类** | 只在菜单中加控件，不涉及游戏调用 | `MenuSwitches.h` → `Menu.cpp` |
| **新 Tab 页** | 在菜单中添加全新页面 | `Menu.h` → `Menu.cpp` → `MenuSwitches.h` |

## 三、每步详细操作 + 文件定位

### 步骤1：从 dump.cs 定位目标类/方法/字段

**目的**：找到你需要调用的 C# 类名、方法名、命名空间、程序集名。

**操作方法**：

1. 用 [Il2CppDumper](https://github.com/Perfare/Il2CppDumper) 对目标游戏生成 `dump.cs`
2. 在 `dump.cs` 中搜索关键词，找到目标类
3. 记录以下 4 个信息：

| 信息 | 在 dump.cs 中的位置 | 示例 |
|:---|:---|:---|
| **程序集名** | 类定义上方的注释或用 `il2cpp_class_get_assemblyname` 验证 | `"Assembly-CSharp"` |
| **命名空间** | `namespace xxx` | `"GameSpecific"` |
| **类名** | `public class Xxx` | `"Player"` |
| **方法名** | `public RetType MethodName()` | `"get_Health"` |

**dump.cs 示例**：

```csharp
namespace GameSpecific                        // ← 命名空间
{
    public class Player : MonoBehaviour       // ← 类名
    {
        private float health;                 // 0x1C  ← 字段偏移
        private float maxHealth;              // 0x20

        public float Health { get; }          // ← 属性

        // RVA: 0x5B120 Offset: 0x5A020
        public float get_Health() { }         // ← IL2CPP 真实方法名
    }
}
```

**验证程序集名的方法**（如果 dump.cs 中没有标注）：

```cpp
// 在 Engine::TestFindObjectsOfType() 中临时添加：
Il2CppClass* pClass = il2cpp_class_from_name(pImage, "GameSpecific", "Player");
if (pClass) {
    printf("Player 程序集: %s\n", il2cpp_class_get_assemblyname(pClass));
}
```

**常见程序集名**：

| 程序集名 | 包含内容 |
|:---|:---|
| `UnityEngine.CoreModule` | Unity 核心类：GameObject, Transform, Object, Camera |
| `UnityEngine.PhysicsModule` | Rigidbody, Collider |
| `Assembly-CSharp` | **游戏主代码**（大多数自定义类在这里） |
| `Assembly-CSharp-firstpass` | 游戏第三方插件 |

### 步骤2：在 il2cpp_function.h 声明 C# 方法

**文件**：`src/Engine/il2cpp_function.h`

**作用**：通过 X-Macro 机制，一行 `DO_FUNC` 声明会自动生成函数指针的类型定义、变量声明、变量定义、运行时绑定。

**语法**：

```cpp
DO_FUNC(返回类型, C++变量名, (参数列表), "程序集名", "命名空间", "类名", "C#方法名")
```

**参数规则**：

| 参数 | 规则 | 示例 |
|:---|:---|:---|
| 返回类型 | 引用类型用指针 `void*`/`Camera*`；值类型用完整定义 `Unity_Vector3`/`float`/`int`/`bool` | `float`, `void*`, `Unity_Vector3` |
| C++变量名 | 自由命名，建议 `类名_方法名` 避免冲突 | `Player_get_Health` |
| 参数列表 | 实例方法第一个参数必须是 `(void*)` 代表 this；静态方法直接写参数 | `(void*)`, `(void*, float)` |
| 程序集名 | 字符串，不带 .dll | `"Assembly-CSharp"` |
| 命名空间 | 字符串，无命名空间写 `""` | `"GameSpecific"` |
| 类名 | 字符串 | `"Player"` |
| C#方法名 | **必须和 IL2CPP 中完全一致**，属性用 `get_`/`set_` 前缀 | `"get_Health"` |

**在文件末尾添加**：

```cpp
// ============================================================================
// Player 常用方法
// ============================================================================

// Player.Health { get; } → 实际方法名: get_Health
DO_FUNC(float, Player_get_Health, (void*),
        "Assembly-CSharp", "GameSpecific", "Player", "get_Health");

// Player.MaxHealth { get; } → 实际方法名: get_MaxHealth
DO_FUNC(float, Player_get_MaxHealth, (void*),
        "Assembly-CSharp", "GameSpecific", "Player", "get_MaxHealth");

// Player.TakeDamage(float amount) → 实例方法，有 this + 一个 float 参数
DO_FUNC(void, Player_TakeDamage, (void*, float),
        "Assembly-CSharp", "GameSpecific", "Player", "TakeDamage");

// Player.GetLocalPlayer() → 静态方法，无 this
DO_FUNC(void*, Player_GetLocalPlayer, (),
        "Assembly-CSharp", "GameSpecific", "Player", "GetLocalPlayer");
```

**C# 属性→方法名对照**：

| C# 代码 | IL2CPP 方法名 |
|:---|:---|
| `float Health { get; }` | `"get_Health"` |
| `float Health { set; }` | `"set_Health"` |
| `float Health { get; set; }` | `"get_Health"` + `"set_Health"` |
| `void TakeDamage(float amt)` | `"TakeDamage"` |
| `static Player GetLocal()` | `"GetLocal"` |

### 步骤3：在 class.h 中定义游戏类型

**文件**：`src/Engine/class.h`

**什么时候需要**：
- 需要通过**硬编码偏移**访问字段时（快但脆弱）
- 需要为 DO_FUNC 提供**返回值类型**时（如 `Camera*`、`Transform*`）
- 需要在 C++ 侧**包装方法**时

**什么时候不需要**：
- 只用 `il2cpp_field_get_value` 动态读取字段
- 只调用 DO_FUNC 绑定的方法指针

**添加位置**：文件末尾

**示例1：为 DO_FUNC 提供类型（简化版）**

```cpp
// class.h 末尾
class Player {
public:
    // 简化版：只声明方法，不定义字段
    float GetHealth();
    float GetMaxHealth();
};
```

**示例2：包含字段偏移的完整版**

```cpp
// class.h 末尾
class Player {
public:
    // 字段偏移常量（从 dump.cs 获取）
    static constexpr size_t HEALTH_OFFSET     = 0x1C;  // float health
    static constexpr size_t MAX_HEALTH_OFFSET = 0x20;  // float maxHealth
    static constexpr size_t TEAM_ID_OFFSET    = 0x24;  // int teamId
    static constexpr size_t IS_DEAD_OFFSET    = 0x28;  // bool isDead

    // 方法
    float GetHealth();
    float GetMaxHealth();
};
```

**在 class.cpp 中实现方法**：

**文件**：`src/Engine/class.cpp`

```cpp
// class.cpp 末尾
float Player::GetHealth()
{
    if (!::Player_get_Health) return 0.0f;
    return ::Player_get_Health(this);
}

float Player::GetMaxHealth()
{
    if (!::Player_get_MaxHealth) return 0.0f;
    return ::Player_get_MaxHealth(this);
}
```

### 步骤4：在 Engine.h/cpp 中添加辅助函数

**文件**：
- 声明：`src/Engine/Engine.h`（在 `namespace Engine` 内）
- 实现：`src/Engine/Engine.cpp`（在 `namespace Engine` 内）

**目的**：将底层 IL2CPP 调用封装为简洁接口，供 ESP 层和 Menu 层使用。

**Engine.h 添加声明**：

```cpp

// ---- Player 信息读取 ----

float GetPlayerMaxHealth(void* pPlayer);
void* GetLocalPlayer();   // 获取本地玩家
```

**Engine.cpp 添加实现**：

```cpp
// 在 namespace Engine { ... } 内，找到 GetPosition 等函数附近

float GetPlayerHealth(void* pPlayer)
{
    if (!pPlayer || !Player_get_Health) return 0.0f;
    return Player_get_Health(pPlayer);
}

float GetPlayerMaxHealth(void* pPlayer)
{
    if (!pPlayer || !Player_get_MaxHealth) return 100.0f;
    return Player_get_MaxHealth(pPlayer);
}

void* GetLocalPlayer()
{
    if (!Player_GetLocalPlayer) return nullptr;
    return Player_GetLocalPlayer();
}
```

#### 7b：在 ESP.cpp 中实现函数

**ESP 函数的标准模板**（使用封装后的新接口，推荐）：

```cpp
// src/ESP/ESP.cpp —— 在 namespace ESP { ... } 内添加

void DrawPlayerHealth()
{
    // ===== 1. 开关检查 =====
    if (!MenuSwitches::g_Toggles.EspPlayerHealth)
        return;

    // ===== 2. 线程安全（RAII 自动管理，无需手动 detach）=====
    Engine::ThreadGuard guard;
    if (!guard.IsValid()) return;

    // ===== 3. 获取主摄像机 =====
    Camera* pCamera = Engine::GetMainCameraSafe();
    if (!pCamera) return;

    // ===== 4. 一行获取 Player 的 System.Type 实例 =====
    // 旧写法需要 6 步链式调用 + 每步 null 检查 + 每步 detach，现在一行搞定
    Il2CppObject* pTypeObj = Engine::GetTypeObject("Assembly-CSharp", "GameSpecific", "Player");
    if (!pTypeObj) return;

    // ===== 5. 调用 FindObjectsOfType 获取所有 Player =====
    if (!FindObjectsOfType) return;
    System_Object_array* players = reinterpret_cast<System_Object_array*>(FindObjectsOfType(pTypeObj));
    if (!players || players->max_length <= 0) return;

    // ===== 6. 遍历 + 绘制 =====
    ImDrawList* drawList = ImGui::GetForegroundDrawList();

    for (size_t i = 0; i < static_cast<size_t>(players->max_length); i++)
    {
        void* pPlayer = players->m_Items[i];
        if (!pPlayer) continue;

        // 获取 Transform → 世界坐标
        void* pTransform = Engine::GetTransform(pPlayer);
        if (!pTransform) continue;

        Unity_Vector3 worldPos = Engine::GetPosition(pTransform);

        // 投影到屏幕
        Vector3 pos3D(worldPos.x, worldPos.y, worldPos.z);
        Vector2 screenPos;

        if (pCamera->WorldToScreen(pos3D, screenPos))
        {
            // ★ 读取玩家血量（通过 Engine 层封装）
            float health = Engine::GetPlayerHealth(pPlayer);
            float maxHealth = Engine::GetPlayerMaxHealth(pPlayer);
            float ratio = (maxHealth > 0) ? health / maxHealth : 0.0f;

            // 绘制血量条
            float barWidth = 40.0f;
            float barHeight = 4.0f;
            float barX = screenPos.X - barWidth / 2;
            float barY = screenPos.Y - 20;

            // 背景（黑色）
            drawList->AddRectFilled(
                ImVec2(barX, barY),
                ImVec2(barX + barWidth, barY + barHeight),
                IM_COL32(0, 0, 0, 180)
            );

            // 血量（颜色根据比例变化）
            ImU32 hpColor = ratio > 0.6f ? IM_COL32(0, 255, 0, 255) :   // 绿色
                            ratio > 0.3f ? IM_COL32(255, 255, 0, 255) :   // 黄色
                                          IM_COL32(255, 0, 0, 255);       // 红色

            drawList->AddRectFilled(
                ImVec2(barX, barY),
                ImVec2(barX + barWidth * ratio, barY + barHeight),
                hpColor
            );

            // 血量数字
            char hpText[32];
            sprintf_s(hpText, "%.0f/%.0f", health, maxHealth);
            drawList->AddText(
                ImVec2(barX, barY - 14),
                IM_COL32(255, 255, 255, 255),
                hpText
            );
        }
    }
    // guard 离开作用域，自动 detach，无需手动处理
}
```

{

    Il2CppThread* pAttachedThread = nullptr;
    if (il2cpp_thread_attach && il2cpp_domain_get) {

    Il2CppDomain* pDomain = il2cpp_domain_get();

**编译**：使用 Visual Studio 打开 `ShootHack.sln`，Build → Build Solution。

**调试技巧**：

1. **函数指针是否绑定成功**：查看控制台输出，`GetMethod` 成功时会打印：
   ```

   失败时会打印：
   ```

   - 确认场景中有该类型的对象

   - 确认 ESP 函数中有 `il2cpp_thread_attach/detach`
   - 确认每个指针访问前都有 null 检查
   - 确认没有在 `Initialize()` 阶段调用需要场景就绪的方法（如 `Camera.get_main()`）

4. **偏移验证**：用 `Engine::DumpObject()` 打印运行时对象的字段列表：

## 六、常见问题排查

### Q1：编译报错 "xxx_t 未定义"

**原因**：`il2cpp_function.h` 中 DO_FUNC 的返回类型或参数类型未在 `class.h` 中定义。

**解决**：在 `class.h` 中添加对应的类型定义。例如 `DO_FUNC(Player*, ...)` 需要 `class.h` 中有 `class Player`。

### Q2：运行时 GetMethod 失败

**排查清单**：

1. 程序集名是否正确？游戏自定义类通常在 `"Assembly-CSharp"`
2. 命名空间是否正确？注意嵌套命名空间如 `"System.Runtime.InteropServices"`
3. 方法名是否带 `get_`/`set_` 前缀？C# 属性 `Health { get; }` → 方法名是 `"get_Health"`
4. 用 `Engine::DumpAssembly("Assembly-CSharp")` 打印所有类，确认类名拼写

### Q3：FindObjectsOfType 返回空数组

**原因**：
- 场景中还没有该类型的对象（游戏还没加载）
- 类名/命名空间错误

**解决**：在 `ESP::DrawPlayerHealth()` 开头加日志：

```cpp
System_Object_array* players = ...;
if (!players || players->max_length <= 0) {
    printf("[*] DrawPlayerHealth: 未找到 Player (count=%zu)\n",
           players ? (size_t)players->max_length : 0);
    // ...
}
```

### Q4：调用方法后崩溃

**排查**：
1. 是否在渲染线程调用了 IL2CPP 方法但没使用 `Engine::ThreadGuard`？
2. 函数指针是否为 nullptr？（检查 Initialize 日志）
3. 实例方法的第一个参数是否传了 this？
4. 参数类型是否匹配？C# 的 `string` → C++ 的 `void*`（Il2CppString*）

### Q5：字段偏移在不同游戏版本中变了

**解决**：用 IL2CPP API 动态获取偏移，而非硬编码：

```cpp
// 脆弱写法：
float health = *reinterpret_cast<float*>((uintptr_t)pPlayer + 0x1C);

// 稳定写法：
static size_t healthOffset = 0;
static bool offsetCached = false;
if (!offsetCached) {
    Il2CppClass* pClass = il2cpp_class_from_name(pImage, "GameSpecific", "Player");
    FieldInfo* pField = il2cpp_class_get_field_from_name(pClass, "health");
    healthOffset = il2cpp_field_get_offset(pField);
    offsetCached = true;
}
float health = *reinterpret_cast<float*>((uintptr_t)pPlayer + healthOffset);
```

### Q6：配置保存/加载不包含新字段

**原因**：配置系统使用 `ConfigSchema.h` 中的宏列表序列化字段，新字段需要在那里也注册。

**解决**：找到 `src/UI/Menu/ConfigSchema.h`，在对应宏列表中添加新字段：

```cpp
// CFG_BOOL_ITEMS 中添加：
CFG_BOOL("EspPlayerHealth", EspPlayerHealth, false, "玩家血量ESP")

// CFG_COLOR_ITEMS 中添加：
CFG_COLOR("ColorPlayerHp", ColorPlayerHp, {0.24f,0.92f,0.33f,1.0f}, "玩家血量颜色")
```
