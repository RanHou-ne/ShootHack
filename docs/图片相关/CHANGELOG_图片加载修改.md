## 🔍 关键发现

### 图片颜色原理

**发现：** 左上角和左下角的 Logo 使用的是**同一个纹理** (`image::logo`)

**位置：**
- 左上角：`Menu.cpp` 第 1186 行（已注释）
- 左下角：`Menu.cpp` 第 1336 行

**代码对比：**
```cpp
// 两处代码几乎相同，只是位置不同
ImGui::GetWindowDrawList()->AddImage(
    image::logo,                     // 同一个纹理！
    pos + ImVec2(...),               // 不同的位置
    pos + ImVec2(...),
    ImVec2(0, 0), ImVec2(1, 1),
    ImColor(255, 255, 255, 255)      // 相同的颜色叠加
);
```

**结论：**
- 如果看起来颜色不同，可能是：
  1. 原图本身有颜色（如橙色）
  2. 背景颜色影响视觉效果
  3. 光照/阴影效果的影响

- 颜色叠加规则：
  - 白色 (255, 255, 255) = 保持原图颜色
  - 其他颜色 = 与原图颜色混合（乘法）
  - Alpha 通道 = 控制透明度

## ⚠️ 注意事项

### 破坏性变更

**移除了文件加载回退逻辑**

- **影响：** 如果 `image.h` 中没有对应的字节数组，图片将无法加载
- **解决：** 确保所有图片都已转换为字节数组并添加到 `image.h`

### 依赖要求

**必须定义 `USE_EMBEDDED_IMAGES` 宏**

- **位置：** `src/Hook/DX11Hook.cpp` 第 27 行
- **代码：** `#define USE_EMBEDDED_IMAGES`
- **影响：** 如果未定义，图片将不会加载
