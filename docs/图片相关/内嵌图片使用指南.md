# 内嵌图片使用指南

## 问题原因

原版的 `image.h` 文件包含内嵌的图片数据（以字节数组形式），但当前代码使用的是从文件加载图片的方式（`LoadTextureFromFileWIC`）。

## 解决方案

我已经添加了从内存加载纹理的功能（`LoadTextureFromMemory`），现在可以使用内嵌的图片数据了。

## 如何启用内嵌图片

### 方法 1：使用预处理器宏（推荐）

1. **在项目设置中添加宏定义**
   
   打开 `ShootHack.vcxproj`，找到 `<PreprocessorDefinitions>` 部分，添加：
   ```xml
   USE_EMBEDDED_IMAGES;%(PreprocessorDefinitions)
   ```

2. **或者在代码中定义**
   
   在 `src/Hook/DX11Hook.cpp` 文件顶部添加：
   ```cpp
   #define USE_EMBEDDED_IMAGES
   ```

3. **重新编译**
   
   编译后，程序会优先使用 `image.h` 中的内嵌图片数据

### 方法 2：直接修改代码（简单但不灵活）

如果你想始终使用内嵌图片，可以删除 `#ifdef USE_EMBEDDED_IMAGES` 条件编译：

```cpp
// 修改前
#ifdef USE_EMBEDDED_IMAGES
extern unsigned char logo[];
logoLoaded = LoadTextureFromMemory(g_pd3dDevice, logo, sizeof(logo), &image::logo);
#endif

// 修改后（直接使用）
extern unsigned char logo[];
logoLoaded = LoadTextureFromMemory(g_pd3dDevice, logo, sizeof(logo), &image::logo);
```

## image.h 中可用的图片

`src/Assets/image.h` 文件包含以下内嵌图片：

1. **`logo[]`** - 主 logo（192x192 PNG）
2. **`roll[]`** - 滚动/箭头图标（192x192 PNG）
3. **`logo_general[]`** - 通用 logo（32x32 PNG）
4. **`arrow[]`** - 箭头图标（14x14 PNG）
5. **`bell_notify[]`** - 通知铃铛（32x32 PNG）
6. **`background_image[]`** - 背景图（JPEG）

## 加载优先级

启用 `USE_EMBEDDED_IMAGES` 后，加载顺序为：

1. **内嵌数据** - 从 `image.h` 加载
2. **文件加载** - 如果内嵌数据加载失败，尝试从文件加载
3. **降级方案** - 如果都失败，使用纯色圆形

## 优势

### 使用内嵌图片的优点：
- ✅ 不需要外部文件
- ✅ 图片数据编译到 DLL 中
- ✅ 部署简单，只需一个 DLL 文件
- ✅ 不会因为文件路径问题导致加载失败

### 使用外部文件的优点：
- ✅ 可以轻松替换图片
- ✅ 不增加 DLL 文件大小
- ✅ 可以动态更新图片

## 技术细节

### LoadTextureFromMemory 函数

新添加的函数可以从内存中的字节数组加载纹理：

```cpp
bool LoadTextureFromMemory(
    ID3D11Device* device,
    const unsigned char* imageData,  // 图片数据指针
    size_t imageSize,                // 数据大小
    ID3D11ShaderResourceView** outSrv // 输出纹理视图
);
```

### 工作原理

1. 使用 WIC (Windows Imaging Component) 创建内存流
2. 从内存流解码图片数据
3. 转换为 RGBA 格式
4. 创建 D3D11 纹理
5. 创建着色器资源视图

### 支持的格式

通过 WIC 支持以下格式：
- PNG
- JPEG
- BMP
- GIF
- TIFF

## 常见问题

### Q: 为什么内嵌图片加载失败？

A: 可能的原因：
1. 没有定义 `USE_EMBEDDED_IMAGES` 宏
2. `image.h` 文件没有被包含
3. 图片数据损坏
4. WIC 组件初始化失败

### Q: 如何添加新的内嵌图片？

A: 步骤：
1. 将图片转换为字节数组（可以使用在线工具或 Python 脚本）
2. 添加到 `image.h` 文件
3. 在加载代码中添加对应的加载逻辑

### Q: 内嵌图片会增加多少 DLL 大小？

A: 取决于图片大小：
- `logo[]` (192x192 PNG): ~50KB
- `roll[]` (192x192 PNG): ~30KB
- 总计约 100-200KB

### Q: 可以同时使用内嵌图片和外部文件吗？

A: 可以！当前实现会先尝试内嵌数据，失败后自动回退到文件加载。

## 推荐配置

- 不定义 `USE_EMBEDDED_IMAGES`

### 发布阶段

- 定义 `USE_EMBEDDED_IMAGES`

## 示例：添加新图片

### 1. 生成字节数组

使用 Python 脚本：
```python
with open('myimage.png', 'rb') as f:
    data = f.read()
    
print('unsigned char myimage[] = {', end='')
for i, byte in enumerate(data):
    if i % 16 == 0:
        print('\n    ', end='')
    print(f'0x{byte:02X}, ', end='')
print('\n};')
```

### 2. 添加到 image.h

```cpp
unsigned char myimage[] = {
    0x89, 0x50, 0x4E, 0x47, ...
};
```

### 3. 在代码中加载

```cpp
extern unsigned char myimage[];
LoadTextureFromMemory(g_pd3dDevice, myimage, sizeof(myimage), &image::myimage);
```
