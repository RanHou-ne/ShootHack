## 概述

本项目使用**内嵌字节数组**的方式加载图片，所有图片资源都编译到 DLL 中，无需外部文件。

### 优势
- ✅ 无需分发图片文件，DLL 自包含
- ✅ 防止用户误删资源文件
- ✅ 加载速度快，无磁盘 IO
- ✅ 便于版本管理

### 工作流程
```
PNG/JPG 图片 → Python 脚本转换 → C++ 字节数组 → 编译到 DLL → 运行时加载
```

## 图片颜色原理

### 🎨 为什么同一张图片显示不同颜色？

**关键点：ImGui 的 `AddImage` 函数支持颜色叠加（Tint Color）**

```cpp
// 函数签名
void AddImage(
    ImTextureID user_texture_id,
    const ImVec2& p_min,
    const ImVec2& p_max,
    const ImVec2& uv_min = ImVec2(0, 0),
    const ImVec2& uv_max = ImVec2(1, 1),
    ImU32 col = IM_COL32_WHITE  // ← 这是颜色叠加参数！
);
```

### 实际案例分析

#### 左上角 Logo（橙色）
```cpp
// 位置：Menu.cpp 第 1186 行（已注释）
ImGui::GetWindowDrawList()->AddImage(
    image::logo,                           // 纹理
    pos + ImVec2(14, 14),                  // 左上角
    pos + ImVec2(46, 46),                  // 右下角
    ImVec2(0, 0),                          // UV 起点
    ImVec2(1, 1),                          // UV 终点
    ImColor(255, 255, 255, 255)            // 白色 = 原图颜色
);
```

#### 左下角 Logo（白色）
```cpp
// 位置：Menu.cpp 第 1336 行
ImGui::GetWindowDrawList()->AddImage(
    image::logo,                           // 同一个纹理！
    pos + ImVec2(...),                     // 不同位置
    pos + ImVec2(...),
    ImVec2(0, 0),
    ImVec2(1, 1),
    ImColor(255, 255, 255, 255)            // 白色 = 原图颜色
);
```

### 颜色叠加规则

| 原图颜色 | 叠加颜色 | 最终效果 |
|---------|---------|---------|
| 橙色 (255, 165, 0) | 白色 (255, 255, 255) | 橙色（保持原样） |
| 橙色 (255, 165, 0) | 红色 (255, 0, 0) | 深红色（橙色×红色） |
| 白色 (255, 255, 255) | 蓝色 (0, 0, 255) | 蓝色 |
| 任意颜色 | 半透明白色 (255, 255, 255, 128) | 半透明原图 |

**结论：如果你的 Logo 原图是橙色，使用白色叠加会保持橙色。如果原图是白色，可以通过叠加任意颜色来改变显示颜色。**

## 准备图片资源

### 推荐规格
- **格式**：PNG（支持透明）或 JPG
- **尺寸**：根据实际需求
  - Logo：64x64 或 128x128
  - 图标：32x32 或 64x64
  - 背景：根据窗口大小
- **颜色模式**：RGB 或 RGBA
- **文件大小**：建议 < 500KB（过大会增加 DLL 体积）

### 示例
```
my_icon.png      → 64x64, RGBA, 15KB
my_background.jpg → 1920x1080, RGB, 300KB
```

## 在代码中加载

### 1. 声明纹理指针

在 `src/Hook/DX11Hook.cpp` 的 `image` 命名空间中添加：

```cpp
namespace image
{
    ID3D11ShaderResourceView* bg           = nullptr;
    ID3D11ShaderResourceView* logo         = nullptr;
    ID3D11ShaderResourceView* logo_general = nullptr;
    ID3D11ShaderResourceView* arrow        = nullptr;
    ID3D11ShaderResourceView* bell_notify  = nullptr;
    ID3D11ShaderResourceView* roll         = nullptr;
    
    // 【新增】你的自定义图标
    ID3D11ShaderResourceView* my_icon      = nullptr;
}
```

### 2. 加载纹理

在 `DX11Hook.cpp` 的 `Present` 函数中，找到纹理加载部分（约 407 行），添加：

```cpp
// 加载 my_icon（强制从内嵌数据加载）
#ifdef USE_EMBEDDED_IMAGES
if (LoadTextureFromMemory(g_pd3dDevice, my_icon, sizeof(my_icon), &image::my_icon))
{
    printf("[+] My Icon 从内嵌数据加载成功\n");
}
else
{
    printf("[!] 错误: My Icon 内嵌数据加载失败！\n");
}
#endif
```

### 3. 添加外部声明

在需要使用图片的文件（如 `Menu.cpp`）顶部添加：

```cpp
namespace image
{
    extern ID3D11ShaderResourceView* bg;
    extern ID3D11ShaderResourceView* logo;
    extern ID3D11ShaderResourceView* roll;
    
    // 【新增】你的自定义图标
    extern ID3D11ShaderResourceView* my_icon;
}
```

## 在 UI 中显示

### 基本语法

```cpp
// 检查纹理是否加载成功
if (image::my_icon)
{
    // 获取当前窗口的绘制列表
    ImDrawList* draw_list = ImGui::GetWindowDrawList();
    
    // 绘制图片
    draw_list->AddImage(
        image::my_icon,              // 纹理指针
        ImVec2(x1, y1),              // 左上角坐标
        ImVec2(x2, y2),              // 右下角坐标
        ImVec2(0, 0),                // UV 起点（通常是 0,0）
        ImVec2(1, 1),                // UV 终点（通常是 1,1）
        ImColor(255, 255, 255, 255)  // 颜色叠加（白色=原图）
    );
}
```

### 坐标系统

```
窗口坐标系：
(0, 0) ────────────────→ X
  │
  │    (x1, y1)
  │      ┌─────────┐
  │      │  图片   │
  │      │         │
  │      └─────────┘
  │           (x2, y2)
  ↓
  Y
```

### 位置计算

#### 方法 1：绝对坐标
```cpp
// 在屏幕 (100, 100) 位置显示 64x64 的图片

```


#### 方法 3：居中显示
```cpp

    pos + ImVec2(centerX + iconSize, centerY + iconSize),
```

## 常见问题

### Q1: 图片不显示怎么办？

**检查清单：**
1. ✅ 确认 `USE_EMBEDDED_IMAGES` 已定义（在 `DX11Hook.cpp` 第 27 行）
2. ✅ 确认字节数组已添加到 `image.h`
3. ✅ 确认纹理指针已声明（`namespace image`）
4. ✅ 确认加载代码已添加（`LoadTextureFromMemory`）
5. ✅ 检查控制台输出是否有加载成功的消息
6. ✅ 确认 `if (image::my_icon)` 条件为真

### Q2: 如何调整图片大小？

```cpp
// 原始大小：64x64
draw_list->AddImage(image::my_icon, pos, pos + ImVec2(64, 64), ...);

// 放大到 128x128
draw_list->AddImage(image::my_icon, pos, pos + ImVec2(128, 128), ...);

// 缩小到 32x32
draw_list->AddImage(image::my_icon, pos, pos + ImVec2(32, 32), ...);

// 拉伸（宽 100，高 50）
draw_list->AddImage(image::my_icon, pos, pos + ImVec2(100, 50), ...);
```

### Q3: 如何实现图片动画？

```cpp
// 淡入淡出效果
static float alpha = 0.0f;
alpha += ImGui::GetIO().DeltaTime * 2.0f;  // 每秒增加 2.0
if (alpha > 1.0f) alpha = 0.0f;            // 循环

draw_list->AddImage(
    image::my_icon,
    pos,
    pos + ImVec2(64, 64),
    ImVec2(0, 0),
    ImVec2(1, 1),
    ImColor(255, 255, 255, (int)(alpha * 255))  // 动态透明度
);
```

### Q4: 如何裁剪图片的一部分？

```cpp
// UV 坐标控制显示图片的哪一部分
// UV (0, 0) = 左上角，UV (1, 1) = 右下角

// 只显示图片的左半部分
draw_list->AddImage(
    image::my_icon,
    pos,
    pos + ImVec2(64, 64),
    ImVec2(0.0f, 0.0f),    // UV 起点：左上角
    ImVec2(0.5f, 1.0f),    // UV 终点：中间到底部
    ImColor(255, 255, 255, 255)
);

// 只显示图片的右下角 1/4
draw_list->AddImage(
    image::my_icon,
    pos,
    pos + ImVec2(64, 64),
    ImVec2(0.5f, 0.5f),    // UV 起点：中心点
    ImVec2(1.0f, 1.0f),    // UV 终点：右下角
    ImColor(255, 255, 255, 255)
);
```

### Q5: 图片太大导致 DLL 体积过大？

**优化建议：**
1. 压缩图片（使用 TinyPNG 等工具）
2. 降低分辨率（UI 图标通常 64x64 足够）
3. 使用 JPG 代替 PNG（如果不需要透明）
4. 考虑使用矢量图标字体（如 Font Awesome）

## 总结

### 快速步骤回顾

1. **准备图片** → PNG/JPG 文件
2. **转换** → `python Scripts/image_to_cpp.py input.png array_name`
3. **添加到 image.h** → 粘贴字节数组
4. **声明指针** → `namespace image { extern ID3D11ShaderResourceView* my_icon; }`
5. **加载纹理** → `LoadTextureFromMemory(...)`
6. **显示图片** → `draw_list->AddImage(...)`

### 关键要点

- ✅ 所有图片必须转换为字节数组
- ✅ 使用 `LoadTextureFromMemory` 加载
- ✅ 颜色叠加参数控制显示效果
- ✅ 坐标系统：左上角 (x1, y1)，右下角 (x2, y2)
- ✅ UV 坐标控制显示图片的哪一部分

### 参考文件
